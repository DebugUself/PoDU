缘起¶

进入 玻璃花园 大家相互观察, 才能感受到自学的参差多态, 是多么欢乐.

背景

一个自学社区面临的典型问题

  • 如何证明自己在自学?
  • 具体都自学了什么?

现象

自怼圈任务丰富

而记录个人自怼证据大多分散在各处分支

收敛自学证据仅集中在手工化的 2 个维度:

Season Plan
时间账单?

为更好得评估日有寸进

需自动化 多维评估工具...

分析

量化分析建立在可观察的外在行为指标上

幸好主要蟒营有效外部活动都在 github 中

而且: API previews | GitHub Developer Guide 也有丰富接口

目标

自动化定期统计关键客观活性数据:

每个成员, 历史/近期 核心行为

  • top5 单独的通证页面, URI 模式:
    • du.101.camp/podu/
  • 包含:
    • commit 次数, 涉及行数
    • commit-comments 次数, 字数
    • issue 次数, 字数, 平均字数/次
    • issue-comments 次数, 字数, 平均字数/次
    • wiki 编辑次数, 涉及字数
  • 以及, 通过归一化, 加权后, 活性指数, 在怼圈历史上的总排名

现状

  • 代码: du4proto/PoDU at DU_tools · DebugUself/du4proto
  • 依赖:
    • Python >= 3.7.0
    • requests
    • numpy
    • matplotlib
    • termgraph
    • textwrap3
  • 统计:
    • ci ~ commit , 自怼仓库 push 次数
    • cc ~ commit-comments , 提交 diff 页面中的回复/讨论次数
    • is ~ issue , 自怼提案创建次数
    • ic ~ issue-comments , 自怼仓提案回复次数

以此综合指数为基础所有成员为数据域进行归一化处置
